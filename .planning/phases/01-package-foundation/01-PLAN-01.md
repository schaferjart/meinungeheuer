# Plan 01: Scaffold package with build tooling, types, and exports map

## Objective

Create the `packages/karaoke-reader/` package directory with all build configuration files, the complete subpath exports map, core type definitions, and placeholder barrel files -- producing a package that builds, typechecks, and passes publint/attw validation with empty or stub content in the Phase 2/3 subpaths.

## Context

- The monorepo already has `packages:` in `pnpm-workspace.yaml` covering `packages/*`, so `packages/karaoke-reader/` is auto-discovered.
- Build tooling: tsup 8.5 for ESM+CJS dual output with `.d.ts` generation, extending `tsconfig.base.json` (strict mode, ES2022, bundler resolution).
- Package must have zero runtime dependencies (PKG-04), React as optional peerDependency (PKG-03).
- The exports map has 5 subpaths: `.` (root), `./utils`, `./hooks`, `./elevenlabs`, `./styles.css`. Hooks and elevenlabs are empty placeholders until Phase 2/3.
- Core types to export: `WordTimestamp`, `AlignmentData`, `TtsStatus`, `LineType`, `ParsedWord`, `ParsedLine`, `ParsedParagraph`.
- If empty barrels cause publint/attw issues, export a trivial placeholder and document it for removal.
