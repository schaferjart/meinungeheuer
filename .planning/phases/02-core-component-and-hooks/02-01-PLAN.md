# Plan 01: Test Infrastructure and Dev Dependencies

## Objective
Set up the testing foundation for React component and hook tests: install `@testing-library/react`, `@testing-library/user-event`, `@testing-library/jest-dom`, `react`, `react-dom`, and `happy-dom` as devDependencies. Configure vitest for DOM environment. Create a reusable `MockAudio` test helper. This is Wave 0 — nothing else can start until this is green.

## Context
Phase 1 left `vitest.config.ts` and test infrastructure for pure utility tests. Phase 2 adds React hooks and a component that manipulate the DOM and use `HTMLAudioElement`. happy-dom's Audio implementation is incomplete, so we need a custom `MockAudio` class. The vitest config needs `environment: 'happy-dom'` for DOM tests. The `tsconfig.json` exclude must also cover `.test.tsx` files.

## Tasks

<task id="01-01">
### Install dev dependencies
Run `pnpm --filter karaoke-reader add -D react react-dom @testing-library/react @testing-library/user-event @testing-library/jest-dom happy-dom @types/react @types/react-dom`. Verify they appear in `package.json` devDependencies.
<verify>
cd packages/karaoke-reader && cat package.json | grep -A5 '"devDependencies"'
</verify>
</task>

<task id="01-02">
### Configure vitest for happy-dom
Update `vitest.config.ts` to set `test.environment: 'happy-dom'`. Ensure `test.globals: true` is set for `@testing-library/jest-dom` matchers. Add setup file if needed for jest-dom `expect` extensions.
<verify>
pnpm --filter karaoke-reader test -- --run 2>&1 | tail -5
</verify>
</task>

<task id="01-03">
### Update tsconfig.json exclude
Add `"**/*.test.tsx"` to the `exclude` array in `packages/karaoke-reader/tsconfig.json` (it already excludes `**/*.test.ts`).
<verify>
pnpm --filter karaoke-reader exec tsc --noEmit
</verify>
</task>

<task id="01-04">
### Create MockAudio test helper
Create `src/test-utils/mock-audio.ts` exporting a `MockAudio` class that extends `EventTarget`. It must have controllable `currentTime`, `duration`, `volume`, `paused`, `readyState`, and methods `play()` (returns Promise), `pause()`, plus helpers `simulateCanPlayThrough()`, `simulateEnded()`, `simulateError()` that dispatch the corresponding events. This avoids reliance on happy-dom's incomplete Audio implementation.
<verify>
pnpm --filter karaoke-reader exec tsc --noEmit --strict src/test-utils/mock-audio.ts
</verify>
</task>

<task id="01-05">
### Wave 0 gate verification
Run the full build + typecheck + exports check + tests to confirm Wave 0 is green.
<verify>
pnpm --filter karaoke-reader build && pnpm --filter karaoke-reader exec tsc --noEmit && pnpm --filter karaoke-reader run check-exports && pnpm --filter karaoke-reader test -- --run
</verify>
</task>
