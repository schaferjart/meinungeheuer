# Plan 04: useKaraokeReader Hook — Orchestrator with Status State Machine

## Objective
Implement the `useKaraokeReader` hook that combines `useAudioSync` + audio lifecycle management + status state machine + volume control + play/pause controls + callbacks into one orchestrator hook. This is the primary API surface for consumers who want headless karaoke functionality.

## Context
This hook composes `useAudioSync` (from Plan 02) and manages the audio element lifecycle. It implements the status state machine (idle -> loading -> ready -> playing -> paused -> done | error). It accepts `audioSrc` as either a URL string (creates Audio internally) or an `HTMLAudioElement` (consumer controls). It handles autoplay-blocked detection (COMP-08), volume control (COMP-06), play/pause toggling (COMP-05), and onComplete callback (COMP-07).

## Tasks

<task id="04-01">
### Implement useKaraokeReader hook
Create `src/hooks/useKaraokeReader.ts`. Accept `{ timestamps, audioSrc, autoPlay?, initialVolume?, onComplete?, onStatusChange?, onError? }`. Return `{ status, activeWordIndex, play, pause, toggle, volume, setVolume, error, audioRef }`.

Implementation:
- Create or wrap `HTMLAudioElement` from `audioSrc` (string -> new Audio(src), HTMLAudioElement -> use directly)
- Status state machine: `idle` (no audioSrc) -> `loading` (audio loading) -> `ready` (canplaythrough) -> `playing` (play()) -> `paused` (pause()) -> `done` (ended). Error reachable from any state.
- Listen to audio events: `canplaythrough` -> ready, `ended` -> done + call onComplete, `error` -> error + call onError
- Autoplay-blocked detection (COMP-08): if `autoPlay` is true and `play()` promise rejects with NotAllowedError, stay in `ready` state (do not go to error)
- Volume: sync `volume` state to `audio.volume`, expose `setVolume`
- `play()`: call `audio.play()`, set status to `playing` on success
- `pause()`: call `audio.pause()`, set status to `paused`
- `toggle()`: play if paused/ready, pause if playing
- Call `onStatusChange` on every status transition
- Cleanup: pause audio, revoke blob URLs if component created them, remove event listeners
- Pass `{ audio, timestamps, enabled: status === 'playing' }` to `useAudioSync`
