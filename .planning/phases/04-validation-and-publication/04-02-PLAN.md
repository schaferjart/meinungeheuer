# Plan 02: README Documentation and npm Publication Preparation

## Objective

Write comprehensive README documentation for the karaoke-reader package and prepare it for npm publication. The README serves as both user documentation and the npm registry landing page. Publication itself requires a manual `npm login` step from the user, so this plan prepares everything up to that point and documents the publish command.

## Context

The karaoke-reader package is feature-complete with 111 tests, correct subpath exports (validated by publint + attw), and a working first consumer (MeinUngeheuer tablet, wired in Plan 01). What remains is user-facing documentation and npm metadata.

The package has 5 subpath exports:
- `karaoke-reader` -- types, utilities, cache factories, hooks, KaraokeReader component
- `karaoke-reader/hooks` -- useAudioSync, useAutoScroll, useKaraokeReader
- `karaoke-reader/utils` -- buildWordTimestamps, splitTextIntoChunks, computeCacheKey, markdown utilities
- `karaoke-reader/elevenlabs` -- fetchElevenLabsTTS, useElevenLabsTTS
- `karaoke-reader/styles.css` -- self-contained CSS with --kr-* custom properties

The README must show two usage modes:
1. **Generic mode**: consumer provides their own timestamps and audio URL
2. **ElevenLabs mode**: consumer provides API key + text, package handles TTS + timestamps

## Tasks

<task id="02-01">
### Add npm metadata to package.json

Update `packages/karaoke-reader/package.json`:

1. Set `"version": "0.1.0"` (initial public release)
2. Remove `"private": true` (unblocks npm publish)
3. Add `"description": "Word-by-word karaoke text highlighting synced to audio playback"`
4. Add `"license": "MIT"`
5. Add `"keywords": ["karaoke", "text-to-speech", "tts", "highlighting", "react", "audio", "elevenlabs", "word-highlighting", "text-reader"]`
6. Add `"author"` field -- use `"MeinUngeheuer Contributors"` as placeholder (the user can update it before publishing)
7. Add `"repository"` field if a public GitHub URL exists
8. Add `"prepublishOnly": "pnpm build && pnpm test && pnpm run check-exports"` to scripts -- ensures quality gate runs before every publish

Do NOT add `"homepage"` or `"bugs"` fields unless a public repo URL is confirmed.

<task id="02-02">
### Write README.md with generic and ElevenLabs usage examples

1. **Title + badge area** -- `# karaoke-reader` with a one-line description
2. **Install** -- `npm install karaoke-reader`

5. **Features** -- bullet list: 60fps word highlighting, auto-scroll, play/pause controls, volume, markdown support, caching, ElevenLabs adapter, CSS custom properties, React 18+19

   - `<KaraokeReader>` props table (14 props from KaraokeReaderProps interface)
