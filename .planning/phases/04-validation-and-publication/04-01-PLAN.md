# Plan 01: Wire MeinUngeheuer Tablet App to Consume karaoke-reader Package

## Objective

Replace the tablet app's local text-reading implementation (useTextToSpeechWithTimestamps hook, TextReader component, ttsCache utilities) with imports from the `karaoke-reader` package. The tablet app becomes the first real consumer of the extracted package, validating that the API surface works for a production use case. VAL-02 and VAL-03 are already satisfied by the 111 existing tests in the karaoke-reader package -- this plan verifies that claim and focuses on the integration work (VAL-01).

## Context

The tablet app currently has ~1200 lines of text-reading code spread across 4 files:
- `useTextToSpeechWithTimestamps.ts` (555 lines) -- monolithic hook with ElevenLabs API, chunking, timestamps, rAF sync, playback
- `TextReader.tsx` (600 lines) -- visual component with markdown parsing, word rendering, highlighting, auto-scroll, controls, i18n
- `ttsCache.ts` (78 lines) -- Supabase-backed TTS cache
- `TextDisplayScreen.tsx` (73 lines) -- thin wrapper passing env vars

All core functionality has been extracted into karaoke-reader. The tablet app keeps its MeinUngeheuer-specific UI (i18n strings, action buttons, loading dots, auto-continue timer, skip button) as a wrapper around `<KaraokeReader>` from the package.

The Supabase cache adapter is tablet-specific and stays in the tablet app, but now implements the generic `CacheAdapter` interface from karaoke-reader.

### Rewrite ttsCache.ts as Supabase CacheAdapter

Rewrite `apps/tablet/src/lib/ttsCache.ts` to export a `createSupabaseTTSCache()` factory function that returns a `CacheAdapter` (from `karaoke-reader`). The adapter wraps the existing Supabase `tts_cache` table queries.

Key differences from old API:
- Old: `getCachedTts(key)` + `storeTtsCache(key, parts, timestamps, textLength, voiceId)` -- separate functions with extra metadata params
- New: `createSupabaseTTSCache()` returns `{ get(key), set(key, value) }` -- implements `CacheAdapter` interface
- The `set` implementation still inserts `text_length` and `voice_id` columns if available, but these are passed via a closure or omitted if the schema allows NULL
- The `computeCacheKey` import changes from local to `karaoke-reader/utils`
- The `WordTimestamp` type import changes from local hook to `karaoke-reader`
- Error handling: `get` returns null on any error, `set` silently swallows errors (fire-and-forget) -- matching both the old behavior and the CacheAdapter contract

```ts
import type { CacheAdapter, TTSCacheValue } from 'karaoke-reader';
import { getSupabaseClient } from './supabase';

export function createSupabaseTTSCache(): CacheAdapter {
  return {
    async get(key: string): Promise<TTSCacheValue | null> {
      try {
        const supabase = getSupabaseClient();
        const { data, error } = await supabase
          .from('tts_cache')
          .select('audio_base64_parts, word_timestamps')
          .eq('cache_key', key)
          .single();
        if (error || !data) return null;
        return {
          audioBase64Parts: data.audio_base64_parts as string[],
          wordTimestamps: data.word_timestamps as TTSCacheValue['wordTimestamps'],
        };
      } catch {
        return null;
      }
    },
    async set(key: string, value: TTSCacheValue): Promise<void> {
      try {
        const supabase = getSupabaseClient();
        const { error } = await supabase.from('tts_cache').insert({
          cache_key: key,
          audio_base64_parts: value.audioBase64Parts,
          word_timestamps: value.wordTimestamps as unknown[],
        });
        if (error && error.code !== '23505') {
          console.warn('[TTS Cache] Write error:', error.message);
        }
      } catch (err) {
        console.warn('[TTS Cache] Write error:', err);
      }
    },
  };
}
```

### Rewrite TextReader.tsx to use KaraokeReader component and useElevenLabsTTS hook

This is the largest task. Rewrite `apps/tablet/src/components/TextReader.tsx` to:

1. Import `KaraokeReader` from `karaoke-reader` and `useElevenLabsTTS` from `karaoke-reader/elevenlabs`
2. Import `stripMarkdownForTTS` from `karaoke-reader/utils`
3. Import `'karaoke-reader/styles.css'` for the component's self-contained CSS
4. Import `TtsStatus` type from `karaoke-reader`
5. Use `useElevenLabsTTS` hook to fetch TTS data (replacing the monolithic `useTextToSpeechWithTimestamps`)
6. Pass the Supabase cache adapter from `createSupabaseTTSCache()`
7. Pass MeinUngeheuer-specific voice settings: `{ stability: 0.35, similarity_boost: 0.65, style: 0.6, use_speaker_boost: true }`
8. Render `<KaraokeReader>` with `hideControls` for the core text + highlighting
9. Keep all MeinUngeheuer-specific UI as wrapper elements around KaraokeReader:
   - i18n strings (German/English)
   - ActionButton sub-component (Continue, Skip, Ready)
   - LoadingDots sub-component
   - VolumeSlider sub-component (tablet's Tailwind-styled version)
   - Auto-continue timer (2s delay after done)
   - Status-dependent control layouts
   - "Tap to pause" hint during playback

The `TextReaderProps` interface stays the same: `{ text, voiceId, apiKey, language, onComplete }`.

Status mapping:
- `useElevenLabsTTS` returns status `'idle' | 'loading' | 'ready' | 'error'` for the TTS fetch phase
- `KaraokeReader`'s `onStatusChange` fires `TtsStatus` for the playback phase (ready, playing, paused, done)
- The wrapper tracks a combined status: loading (TTS fetching), ready (TTS ready but not playing), playing, paused, done, error

The wrapper component manages the combined status by:
1. While `useElevenLabsTTS` status is `loading` -> show loading UI
2. While `useElevenLabsTTS` status is `error` -> show error UI
3. When `useElevenLabsTTS` status is `ready`, render `<KaraokeReader>` and track its `onStatusChange`
4. Playback status (playing/paused/done) comes from KaraokeReader's callbacks
