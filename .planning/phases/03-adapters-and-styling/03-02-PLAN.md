# Plan 02: ElevenLabs TTS Adapter — fetchElevenLabsTTS and useElevenLabsTTS

## Objective
Implement the optional ElevenLabs adapter as a separate subpath export (`karaoke-reader/elevenlabs`) that fetches TTS audio with character-level timestamps, converts them to `WordTimestamp[]` via `buildWordTimestamps`, handles text chunking for long inputs, integrates the `CacheAdapter` from Plan 01, and provides both a plain async function (`fetchElevenLabsTTS`) and a convenience React hook (`useElevenLabsTTS`).

## Context
The adapter entry point at `src/adapters/elevenlabs/index.ts` is currently a placeholder (`export {}`). The tsup config already has a separate entry point for it, and `package.json` exports `./elevenlabs` correctly. The build already isolates this code from the root bundle (tree-shakeable by design).

The reference implementation lives at `apps/tablet/src/hooks/useTextToSpeechWithTimestamps.ts` (555 lines). We extract only the TTS fetching logic — API call, base64 decoding, chunking orchestration, cache integration. Playback/sync/DOM logic already lives in the package hooks.

Key integration points:
- `buildWordTimestamps(chunkText, alignment, timeOffset)` from `../../utils/buildWordTimestamps.js`
- `splitTextIntoChunks(text, maxWords)` from `../../utils/splitTextIntoChunks.js`
- `computeCacheKey(text, voiceId)` from `../../utils/computeCacheKey.js`
- `CacheAdapter`, `TTSCacheValue`, `WordTimestamp`, `AlignmentData` from `../../types.js`

The API endpoint is `POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}/with-timestamps`. Response is JSON with `audio_base64`, `alignment`, `normalized_alignment`. Prefer `alignment` over `normalized_alignment`.

<task id="02-04">
### Implement fetchElevenLabsTTS orchestration function
Export `fetchElevenLabsTTS(options: ElevenLabsTTSOptions): Promise<ElevenLabsTTSResult>`. Implementation flow:

1. Compute cache key via `const cacheKey = await computeCacheKey(text, voiceId)` — note `computeCacheKey` is async (returns `Promise<string>`) since it uses `crypto.subtle.digest`; the `await` is required
2. If `cache` provided, try `cache.get(cacheKey)` — on hit, decode `audioBase64Parts` to blob URL, return `{ audioUrl, timestamps: cached.wordTimestamps }`
3. On cache miss: split text via `splitTextIntoChunks(text, maxWordsPerChunk)`
4. For each chunk sequentially:
   a. Call `fetchChunkTTS` with chunk text
   b. Use `alignment` if non-null; otherwise fall back to `normalized_alignment`. (Task 02-03 step 7 already validates that at least one is present, so this fallback is safe.)
   c. Call `buildWordTimestamps(chunkText, alignment, timeOffset)` with accumulated time offset
   d. Update `timeOffset` to the last character's `character_end_times_seconds` value
   e. Reindex word timestamps by adding the total word count from prior chunks
   f. Collect `audio_base64` parts and word timestamps
5. Create blob URL from all base64 parts via `base64PartsToAudioUrl`
6. If `cache` provided, fire-and-forget `cache.set(cacheKey, { audioBase64Parts, wordTimestamps })` — wrap in try/catch
7. Return `{ audioUrl, timestamps }`

<task id="02-05">
### Implement useElevenLabsTTS convenience hook
Export `useElevenLabsTTS(options: ElevenLabsTTSOptions | null)` returning `{ status, result, error }`. Implementation:
1. Uses `useState` for `status: 'idle' | 'loading' | 'ready' | 'error'`, `result: ElevenLabsTTSResult | null`, `error: Error | null`
2. `useEffect` watches `options` — when non-null, creates `AbortController`, sets status to `loading`, calls `fetchElevenLabsTTS` with the signal
3. On success: sets status to `ready`, result to the return value
4. On error: if abort, ignores; otherwise sets status to `error`, error to the caught error
5. Cleanup: aborts in-flight request, revokes any blob URL from previous result
6. Returns `null` options = idle state (no fetch)

The hook does NOT import any DOM or audio APIs directly — it only wraps the `fetchElevenLabsTTS` function.
