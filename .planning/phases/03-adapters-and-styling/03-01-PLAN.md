# Plan 01: Cache Layer — CacheAdapter Interface and Built-in Adapters

## Objective
Define the generic `CacheAdapter` and `TTSCacheValue` types, implement `createMemoryCache()` and `createLocalStorageCache()` factory functions, export them from the root package, and test fire-and-forget error semantics. This unblocks Plan 02 (ElevenLabs adapter) which accepts an optional `CacheAdapter`.

## Context
The research doc (Section 3) specifies an async `CacheAdapter` interface with `get(key): Promise<TTSCacheValue | null>` and `set(key, value): Promise<void>`. The memory cache wraps a `Map`, the localStorage cache wraps `JSON.parse/stringify` with error swallowing. Both must never throw — cache errors are silently ignored so playback is never blocked. The existing `computeCacheKey` utility already generates SHA-256 keys; the cache layer consumes those keys.

The `TTSCacheValue` shape is `{ audioBase64Parts: string[], wordTimestamps: WordTimestamp[] }` — matching the reference `ttsCache.ts` in the tablet app.

## Tasks

<task id="01-01">
### Add CacheAdapter and TTSCacheValue types to types.ts
Append to `src/types.ts`:
```ts
export interface TTSCacheValue {
  audioBase64Parts: string[];
  wordTimestamps: WordTimestamp[];
}

export interface CacheAdapter {
  get(key: string): Promise<TTSCacheValue | null>;
  set(key: string, value: TTSCacheValue): Promise<void>;
}
```
These are general-purpose types, not adapter-specific, so they belong in the root types file.
<verify>
pnpm --filter karaoke-reader exec tsc --noEmit
</verify>
</task>

<task id="01-02">
### Implement createMemoryCache
Create `src/cache.ts` with `createMemoryCache()`. Uses an internal `Map<string, TTSCacheValue>`. Both `get` and `set` are async (return promises) for interface consistency. Wrap in try/catch to guarantee fire-and-forget — though a Map should never throw, the contract says cache errors never propagate.
<verify>
pnpm --filter karaoke-reader exec tsc --noEmit
</verify>
</task>

<task id="01-03">
### Implement createLocalStorageCache
Add `createLocalStorageCache(prefix?: string)` to `src/cache.ts`. Default prefix is `'kr-tts-'`. `get` calls `localStorage.getItem`, parses JSON, returns `null` on any error (missing key, invalid JSON, localStorage unavailable). `set` calls `localStorage.setItem` with `JSON.stringify`, silently catches `QuotaExceededError` and any other exceptions.
<verify>
pnpm --filter karaoke-reader exec tsc --noEmit
</verify>
</task>

<task id="01-04">

Add to `src/index.ts`:

```ts
export { createMemoryCache, createLocalStorageCache } from './cache.js';
```
Verify build produces correct declarations and the exports resolve.

</task>

<task id="01-05">
### Write cache adapter unit tests
Create `src/cache.test.ts` with the following tests:

**memoryCache:**
1. `get` returns `null` for unknown key
2. `set` then `get` returns the stored value
3. Two separate `createMemoryCache()` instances have isolated stores

**localStorageCache:**
4. `get` returns `null` for unknown key
5. `set` then `get` returns the stored value
6. `get` returns `null` when `localStorage.getItem` throws (e.g., SecurityError in private browsing)
7. `get` returns `null` when stored data is invalid JSON (corrupted entry)
8. `set` does not throw when `localStorage.setItem` throws (quota exceeded)
9. Custom prefix is applied to localStorage keys

All tests must verify fire-and-forget semantics: no promise rejections, no thrown errors.
<verify>
pnpm --filter karaoke-reader test -- --run src/cache.test.ts
</verify>
</task>
