  <name>Task 2: Add pnpm scripts and verify POS server starts</name>

  <action>
Add three scripts to the root `package.json` `scripts` object:

```json
"dev:pos": "cd apps/pos-server && ./venv/bin/python print_server.py --dummy",
"start:pos": "cd apps/pos-server && ./venv/bin/python print_server.py",
"setup:pos": "cd apps/pos-server && bash setup.sh --deps-only"
```

The `--dummy` flag on `dev:pos` runs the POS server without a physical printer (logs prints to console). The `setup:pos` script creates the Python venv and installs dependencies from `requirements.txt`.

After adding scripts, run the setup and verify:
1. Run `cd apps/pos-server && bash setup.sh --deps-only` to create venv and install deps
2. Verify Flask is installed: `apps/pos-server/venv/bin/python -c "import flask; print(flask.__version__)"`
3. Start the server in dummy mode in background: `cd apps/pos-server && ./venv/bin/python print_server.py --dummy &`
4. Wait 2 seconds, then `curl -s http://localhost:9100/health` â€” expect `{"status":"running","dummy":true}`
5. Kill the background server process
6. Test `/print/dictionary` endpoint: start server again, then `curl -s -X POST http://localhost:9100/print/dictionary -H "Content-Type: application/json" -d '{"word":"TEST","definition":"A verification test."}'` â€” expect `{"status":"ok","template":"dictionary"}`
7. Kill server

Note: If `setup.sh` has issues with `--deps-only` flag, inspect the script and create the venv manually: `cd apps/pos-server && python3 -m venv venv && ./venv/bin/pip install -r requirements.txt`.
