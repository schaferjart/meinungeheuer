**Step A: Extend the state machine** (`useInstallationMachine.ts`)

1. Add `stages` to `InstallationState`:
```typescript

```

Default `stages` in initialState: `{ textReading: true, termPrompt: false, portrait: true, printing: true }` (matches current text_term behavior).

2. Extend `SET_CONFIG` action to include `stages: StageConfig`:
```typescript
| { type: 'SET_CONFIG'; mode: Mode; term: string; contextText: string | null; parentSessionId: string | null; stages: StageConfig }
```

3. Replace mode string comparisons in the reducer with stage booleans:

`TIMER_3S` case (welcome -> next):
- OLD: `if (state.mode === 'text_term' || state.mode === 'chain') return text_display; else return term_prompt;`
- NEW: `if (state.stages.textReading) return text_display; else if (state.stages.termPrompt) return term_prompt; else return conversation;`

`READY` case (text_display -> next):
- OLD: `if (state.mode === 'text_term') return conversation; else return term_prompt;`
- NEW: `if (state.stages.termPrompt) return term_prompt; else return conversation;`

IMPORTANT: The reducer must NOT reference program IDs. It only uses `state.stages` booleans. This is the key anti-pattern to avoid.

4. Update `SET_CONFIG` handler to store stages: `stages: action.stages`.
