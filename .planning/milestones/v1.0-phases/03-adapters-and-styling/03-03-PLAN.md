# Plan 03: CSS Styling â€” Custom Properties and Styled Defaults

## Objective
Populate `src/styles.css` with a self-contained CSS stylesheet that makes `<KaraokeReader>` look polished out of the box (Georgia serif, dark background, amber highlight, smooth transitions) with zero Tailwind dependency. All visual properties are overridable via `--kr-*` CSS custom properties set on a parent element.

## Context
The KaraokeReader component already renders all necessary CSS class names and data attributes:
- `.kr-root` with `data-kr-status` attribute
- `.kr-scroll-container`, `.kr-content`, `.kr-paragraph`, `.kr-line`, `.kr-line--header`, `.kr-line--list-item`, `.kr-line--text`
- `.kr-word` with `data-kr-state` attribute (`upcoming`, `active`, `spoken`) and optional `.kr-word--strikethrough`
- `.kr-loading-indicator`, `.kr-error-fallback`
- `.kr-controls`, `.kr-volume-label`, `.kr-volume-slider`

The tsup build already copies `src/styles.css` to `dist/styles.css` via the `onSuccess` hook. The package.json exports `./styles.css` correctly. No build config changes needed.

This plan is independent of Plans 01 and 02 (cache and adapter). It can execute in parallel with Plan 02 since it touches only `src/styles.css`, which no other plan modifies.

The visual design reference comes from the existing MeinUngeheuer TextReader component (inline styles and Tailwind classes). The research doc (Section 5) specifies all custom property names, defaults, and the three-layer CSS architecture.
