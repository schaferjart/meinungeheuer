# Plan 05: KaraokeReader Component — Rendering, DOM Sync, Controls

## Objective
Implement the `<KaraokeReader>` React component that renders parsed markdown text as word spans, syncs `data-kr-state` attributes to audio playback via direct DOM manipulation at 60fps, integrates auto-scroll, and provides built-in playback controls (tap-to-pause, keyboard, volume slider, loading/error states).

## Context
This component composes `useKaraokeReader` (Plan 04) and `useAutoScroll` (Plan 03). It renders text parsed by `parseMarkdownText` (Phase 1 utility) into paragraphs -> lines -> word `<span>` elements. Each span gets `data-kr-index` and `data-kr-state` attributes. A `useEffect` reads `activeWordIndex` and directly manipulates DOM `data-kr-state` attributes (proven 60fps pattern from TextReader.tsx). Word spans are tracked via a `Map<number, HTMLSpanElement>` ref populated by ref callbacks. Built-in controls (volume slider, status-dependent buttons) can be hidden via `hideControls` prop.

## Tasks

<task id="05-01">
### Implement KaraokeReader component
Create `src/components/KaraokeReader.tsx`. Accept `KaraokeReaderProps` (as specified in research: text, timestamps, audioSrc, autoPlay, onComplete, onStatusChange, onError, initialVolume, hideControls, scrollComfortTop, scrollComfortBottom, scrollCooldown, className, style).

Implementation:
- Parse `text` via `parseMarkdownText` into paragraphs/lines/words
- Render word spans: `<span data-kr-index={globalIndex} data-kr-state="upcoming" ref={callback}>{word}</span>`
- Word ref map: `Map<number, HTMLSpanElement>` populated by ref callbacks during render
- DOM sync `useEffect`: on `activeWordIndex` change, update `data-kr-state` on previous word (-> "spoken"), current word (-> "active"). Use `prevActiveRef` pattern.
- Integrate `useAutoScroll` with containerRef and wordElementsRef
- Tap/click on scroll container: call `toggle()` (COMP-05)
- Keyboard: `onKeyDown` handler on root div for Space/Enter -> `toggle()` (COMP-05), root div has `tabIndex={0}` and `role="application"`
- Volume slider sub-component: range input, syncs to `setVolume` (COMP-06)
- Loading state: render loading indicator when status is `loading` (COMP-08)
- Error state: render error fallback with error message when status is `error` (COMP-08)
- CSS class names use `kr-` prefix (e.g., `kr-root`, `kr-scroll-container`, `kr-content`, `kr-controls`, `kr-word`)
