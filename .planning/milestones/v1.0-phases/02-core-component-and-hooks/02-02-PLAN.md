# Plan 02: useAudioSync Hook — rAF Loop with Binary Search

## Objective
Implement the `useAudioSync` hook that synchronizes an `activeWordIndex` to an `HTMLAudioElement`'s `currentTime` at 60fps using binary search over sorted `WordTimestamp[]`. This is the performance-critical inner loop of the karaoke reader.

## Context
The existing `useTextToSpeechWithTimestamps.ts` contains this logic at lines 272-309 (`updateActiveWord()` with binary search and rAF loop). Extract it as a standalone hook. The hook outputs `activeWordIndex` as React state (proven pattern — cheap re-renders since word spans don't change in the React tree). The rAF loop starts when `enabled=true` and stops when `enabled=false`. Binary search finds the word whose `[startTime, endTime]` bracket contains `audio.currentTime`.

## Tasks

<task id="02-01">
### Implement useAudioSync hook
Create `src/hooks/useAudioSync.ts`. Accept `{ audio: HTMLAudioElement | null, timestamps: WordTimestamp[], enabled: boolean }`. Return `{ activeWordIndex: number }`. Implementation: when `enabled` is true and `audio` is non-null, start a rAF loop that reads `audio.currentTime`, runs binary search over `timestamps`, and calls `setActiveWordIndex` only when the index changes. Cancel rAF on cleanup or when `enabled` becomes false. Return -1 when no word is active.

### Write unit and integration tests
Create `src/hooks/useAudioSync.test.ts`. Tests:
1. Returns -1 initially when no audio or timestamps empty
2. Binary search correctness: given timestamps and a currentTime, returns correct index
3. Gap handling: currentTime between words returns previous word's index
4. Edge cases: currentTime before first word (-1), after last word (last index)
5. Updates activeWordIndex when audio.currentTime advances (use MockAudio + fake timers to advance rAF)
6. Stops updating when enabled=false
7. Cleans up rAF on unmount

Use `renderHook` from `@testing-library/react`, `MockAudio` from test-utils, and `vi.useFakeTimers()` for rAF control.
