# MeinUngeheuer — Product Requirements Document
### Glossary Engine Installation | v0.3

---

## 1. CONCEPT

An interactive installation that produces a personalized glossary through spoken dialogue. A visitor is drawn into conversation with an AI agent about a concept. The AI challenges them to articulate their own understanding — their metaphors, contradictions, examples, emotional weight. The conversation is distilled into a printed definition: an aphorism in the visitor's own voice.

**Reference:** Isaac Asimov, *Words of Science* — where every scientific term is defined not by the dictionary but through etymology, analogy, and personal perspective.

**Philosophical foundation:** Heinrich von Kleist, *Über die allmähliche Verfertigung der Gedanken beim Reden* (1805) — the thesis that thought forms through the act of speaking it. The installation is a machine that performs Kleist's thesis: the visitor does not arrive with a definition; they discover it by talking.

**Core principle:** Genuine understanding begins when someone can express a concept in their own vocabulary. The installation treats this as *appropriation* — making an idea yours by finding your own words for it.

**Output:** A thermal-printed glossary card (term + personalized definition + citations from the visitor's own speech).

## 2. THREE MODES

### Mode A: Text → Term

1. A text is displayed on the tablet (and read aloud)
2. The visitor reads/listens
3. The AI picks a concept from the text and challenges the visitor's comprehension
4. Conversation → Definition → Print

**Context text for MVP:** An excerpt from Kleist's *Über die allmähliche Verfertigung der Gedanken beim Reden*. The visitor reads Kleist's thesis about thought forming through speech — then is asked to do exactly that. The text becomes both content and instruction.

**Why:** The text provides shared ground — a mutual context between machine and human. The challenge: *You just encountered this idea. Now make it yours.*

### Mode B: Term Only

1. A term is displayed: e.g., "WHAT IS A BIRD?"
2. Conversation → Definition → Print

**Why:** Simpler, faster, no reading. Good for testing and high-traffic moments.

### Mode C: Chain Mode ★

The most artistically powerful mode. **Each visitor's definition becomes the context for the next visitor.**

1. Visitor 1 receives a term (e.g., "BIRD") → converses → produces a definition
2. Visitor 2 **reads Visitor 1's definition** as their text context
3. The AI picks a term or concept *from Visitor 1's definition* and challenges Visitor 2 on it
4. Visitor 2's definition becomes context for Visitor 3
5. And so on, indefinitely

**What emerges:** A drift through conceptual space. BIRD → FLIGHT → FREEDOM → CAGE → STRUCTURE → CONTROL → ... Each step determined by how the previous person thinks. Over an exhibition day: a collective path through ideas, generated by strangers.

**Archive value:** This mode produces the richest data for semantic analysis. You can literally trace how thought migrates through multidimensional word-embedding space across hundreds of visitors.

**Implementation:** The chain state is stored in Supabase. After each session, the generated definition is saved. The next session loads the most recent definition as its `context_text`, and the AI is instructed to identify a compelling concept within it.

**Seed:** The chain needs a starting point. Options:
- First visitor of each day gets Mode B (naked term)
- First visitor gets Kleist excerpt (Mode A) → then chain continues from there
- Manual reset by operator at any point

**All three modes produce the same output format** (glossary card) and feed into the same archive. The operator (Jonas) switches modes via config.

## 4. COMPONENT DETAILS

### 4.1 WAKE / SLEEP

**Purpose:** Detect when someone is ready to interact. Detect when they've left.

```
INPUT:  Tablet front-facing camera
OUTPUT: WAKE | SLEEP events
```

- WAKE: Face detected for ~3 seconds (MediaPipe Face Detection, runs in-browser) OR tap-to-start
- SLEEP: No face for 30 seconds after session ends
- No identification. No facial data stored. People demark themselves with their thoughts.
- Fallback: physical tap-to-start button on screen

### 4.2 FRONTEND (UI)

**Framework:** React web app (runs in tablet browser).

**Input:**
- Screen: touch (for wake, mode selection, "tap when ready" after reading)
- Microphone: **external mic with 3.5mm jack** (directional, better audio than tablet built-in)
- Speaker: tablet built-in or external via 3.5mm/Bluetooth

**Screen States:**

| # | State | Screen | Audio | Duration |
|---|-------|--------|-------|----------|
| 0 | SLEEP | Black / breathing animation | Silent | Until wake |
| 1 | WELCOME | "Nähern Sie sich." / "Come closer." | Chime (optional) | 3s |
| 2 | TEXT_DISPLAY | Text appears on screen. Scrollable. "Tap when ready" at bottom. *(Mode A + C only)* | Text read aloud via TTS (optional) | Variable (visitor pace) |
| 3 | TERM_PROMPT | Term displayed large. AI asks first question via voice. | ElevenLabs TTS | 5s |
| 4 | CONVERSATION | Live conversation. Visitor's transcript appears (light gray). AI's words appear as typewriter text synced to voice. Loop until agent decides to synthesize. | Mic active ↔ TTS | Variable (2-7 min) |
| 5 | SYNTHESIZING | "Einen Moment..." / "One moment..." | Silent | 3-8s |
| 6 | DEFINITION | Generated definition displayed (same typography as printed card) | Optional: read aloud | 10s |
| 7 | PRINTING | "Ihr Beitrag wird gedruckt." | Printer sounds | 5-10s |
| 8 | FAREWELL | "Danke." / "Thank you." | Silent | 15s → SLEEP |

**Mode A/C:** States 0-1-2-3-4-5-6-7-8
**Mode B:** States 0-1-3-4-5-6-7-8 (skip text display)

**UI Design:**
- Black background, white text, minimal
- Large typography (readable from 1m)
- Text display (state 2): clean reading layout, generous line-height
- Visitor transcript: smaller, lighter gray, real-time
- AI text: larger, white, typewriter effect synced to audio
- No buttons during conversation (voice-only after initial tap)

### 4.3 ELEVENLABS AGENT CONFIGURATION

The ElevenLabs agent is configured with:

**Voice:** A warm, neutral voice. Test German + English voices from their library. One consistent voice per installation run.

**LLM:** Custom LLM pointing to OpenRouter (see 3.3).

**System tools enabled:**
- `language_detection` — auto-switch language when visitor switches
- `end_conversation` — agent calls this when synthesis is complete

**Custom tool (webhook):**
- Name: `save_definition`
- Trigger: Agent calls this when it has synthesized the definition
- Payload: `{term, definition_text, citations[], language, session_metadata}`
- Endpoint: Cloud backend URL
- This tool replaces `end_conversation` — when the agent calls `save_definition`, the backend saves the definition and triggers printing, and ElevenLabs ends the conversation

**System Prompt:**

```
You are an interviewer in an art installation called "MeinUngeheuer."

{{#if context_text}}
A visitor has just read the following text:
---
{{context_text}}
---
The concept you will explore with them is: {{term}}
This concept appeared in the text they just read.
{{else}}
You will explore the concept: {{term}}
{{/if}}

YOUR GOAL is to understand how THIS PERSON thinks about {{term}}.
Not the textbook definition. Not what they think you want to hear.
Their personal understanding — their metaphors, examples, contradictions,
emotional associations, the way they connect this idea to their own life.

You believe that genuine understanding begins when someone can express
a concept in their own words. You are looking for their unique voice.

RULES:
1. Ask ONE question at a time. 1-2 sentences max.
2. Match the visitor's language. If they switch, you switch.
3. Never lecture. Never define the term yourself. Never correct them.
4. Surface-level answer? Push deeper: ask for an example, a memory,
   a comparison.
5. Something original or revealing? Acknowledge briefly and go deeper.
6. They ask you to explain? Brief factual note, then redirect:
   "But what matters is how YOU think about it."
7. "I don't know" → "What comes to mind first?"
   or "What does the word itself sound like to you?"

STOP WHEN:
- They've expressed a genuinely personal or original perspective
- You've found a productive contradiction or unexpected connection
- They're repeating themselves
- They seem uncomfortable or want to stop
- You've had 5-7 exchanges

WHEN STOPPING:
Call the save_definition tool with:
- term: "{{term}}"
- definition_text: A 2-3 sentence definition that sounds like something
  THEY would say. Preserve their metaphors. Capture what makes their
  view distinctive. Make it memorable — worth printing and keeping.
- citations: 2-3 direct quotes from what they said that were most
  revealing or original. Use their exact words.
- language: "de" or "en" (whichever they spoke)

TONE: Warm, precise, genuinely curious. Like the best conversation
at a dinner party — someone who makes you think harder by listening well.

EDGE CASES:
- Silence > 15s: "Take your time. There's no right answer."
- Dictionary answer: "That's the official version. But what does it
  mean to you?"
- They want to stop: Immediately call save_definition with what you have.
- Trolling: Treat any answer as genuine. If truly nonsensical after
  2 attempts, synthesize gracefully.
- They ask what this is: "I'm curious how you think about {{term}}.
  There are no wrong answers."
```

**Mode C addition to system prompt:**

When in chain mode, the `context_text` is the previous visitor's definition. Add to the prompt:

```
The text above was written by a previous visitor to this installation.
It represents how they think about a concept. Now it's your turn to
find a concept WITHIN their words that is worth exploring with the
current visitor.

Pick ONE concept from the previous visitor's text that:
- Is rich enough to have multiple interpretations
- Would reveal something about how the current visitor thinks
- Naturally continues the thread of ideas

State the concept clearly, then begin your conversation about it.
```

**Printer Bridge:**
- Runs locally (tablet, Pi, or laptop)
- Subscribes to Supabase `print_queue` table via Supabase Realtime
- On new row: reads payload → formats ESC/POS → sends to printer → updates status
- Heartbeat every 30s to verify printer is connected

### 4.5 DATA STORE (Supabase)

**Schema:**

```sql
-- Sessions
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at TIMESTAMPTZ DEFAULT now(),
  ended_at TIMESTAMPTZ,
  mode TEXT NOT NULL CHECK (mode IN ('text_term', 'term_only', 'chain')),
  term TEXT NOT NULL,
  context_text TEXT,                 -- The text shown to visitor (Mode A/C)
  parent_session_id UUID REFERENCES sessions(id),  -- Mode C: which session preceded this one
  language_detected TEXT,
  duration_seconds INTEGER,
  turn_count INTEGER,
  card_taken BOOLEAN,
  elevenlabs_conversation_id TEXT,   -- ElevenLabs session reference
  audio_url TEXT                     -- Debug phase only
);

-- Dialogue turns
CREATE TABLE turns (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
  turn_number INTEGER NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('visitor', 'agent')),
  content TEXT NOT NULL,
  language TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Generated definitions (the output / the art)
CREATE TABLE definitions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id) UNIQUE,
  term TEXT NOT NULL,
  definition_text TEXT NOT NULL,
  citations TEXT[],                  -- Direct quotes from visitor
  language TEXT NOT NULL,
  chain_depth INTEGER DEFAULT 0,    -- Mode C: how many links deep
  created_at TIMESTAMPTZ DEFAULT now(),
  embedding VECTOR(1536)            -- For semantic analysis
);

-- Print queue
CREATE TABLE print_queue (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id),
  payload JSONB NOT NULL,
  printer_config JSONB,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'printing', 'done', 'error')),
  created_at TIMESTAMPTZ DEFAULT now(),
  printed_at TIMESTAMPTZ
);

-- Chain state (Mode C: what's the current context for the next visitor?)
CREATE TABLE chain_state (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  definition_id UUID REFERENCES definitions(id),
  active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

**Semantic Analysis (post-exhibition):**

The `embedding` column enables:
- **Clustering:** How do different visitors' definitions of "BIRD" group in semantic space?
- **Chain drift:** In Mode C, trace the path of thought through embedding space across a day's visitors. Visualize as a trajectory.
- **Cross-pollination:** Find visitors whose definitions of different terms land near each other — unexpected conceptual bridges.
- **Word2Vec proximity:** Map the relationships between terms as defined by visitors vs. as defined by standard word embeddings. Where do people diverge from the machine?

**Embedding generation:** OpenAI `text-embedding-3-small` (1536 dimensions). Generated async after each session by the cloud backend.

## 6. CONTENT

### MVP Term: BIRD

"What is a bird?"

Everyone knows. Nobody has thought about it. The personal definition reveals everything about how someone connects ideas — biology, freedom, childhood, sound, evolution, fear, beauty. Asimov-grade.

### Mode A Text: Kleist Excerpt

Use the opening section of *Über die allmähliche Verfertigung der Gedanken beim Reden*:

> "Wenn Du etwas wissen willst und es durch Meditation nicht finden kannst, so rathe ich Dir, mein lieber, sinnreicher Freund, mit dem nächsten Bekannten, der dir aufstößt, darüber zu sprechen. Es braucht nicht eben ein scharfdenkender Kopf zu sein, auch meine ich es nicht so, als ob du ihn darum befragen solltest, nein! Vielmehr sollst Du es ihm selber allererst erzählen."

Kleist's core insight: *"Die Idee kommt beim Sprechen."* The idea comes through speaking. The visitor reads this, then is asked to do exactly that.

**Terms to extract from Kleist:** VERFERTIGUNG, GEDANKE, MEDITATION, SPRECHEN, BELEHREN

### Mode C Seed

Day begins with Mode B (term: BIRD) or Mode A (Kleist). After first visitor, switch to Mode C. Chain runs for the rest of the day.
